# Bible Bot

Telegram-бот присылает один отобранный стих или короткий отрывок из Нового
Завета каждый день в выбранное пользователем время. Используется локальный
текст Синодального перевода: после установки бот не обращается к внешним
библейским API.

## Пользовательский сценарий

1. `/start` показывает короткое приветствие.
2. Пользователь оставляет время `09:00, Europe/Minsk` или выбирает своё.
3. Первый стих приходит сразу, первая плановая отправка — только на следующий
   день.
4. Под стихом доступны контекст, сохранение и настройки.
5. Время, часовой пояс, паузу и отключение можно изменить в любой момент.
6. После последнего дня цикла бот отправляет Откровение 22:21, показывает итог
   и ставит рассылку на паузу.
7. Дальше можно начать новый круг, идти по порядку книг или выбрать тему.

Основной цикл содержит 365 уникальных отобранных отправок. В общем режиме стих
дня одинаков для всех, но приходит в локальное время каждого пользователя.
Тематические циклы включают веру, надежду, любовь, молитву, слова Иисуса,
утешение и отношения с людьми.

## Команды

- `/today` — получить сегодняшний стих;
- `/settings` — изменить расписание или состояние рассылки;
- `/favorites` — открыть сохранённые стихи;
- `/pause` — поставить рассылку на паузу;
- `/help` — показать справку.

## Локальный запуск

Нужен Python 3.11 или новее и токен бота от
[@BotFather](https://t.me/BotFather).

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -e '.[dev]'
cp .env.example .env
```

Заполните `BOT_TOKEN` в `.env`, затем:

```bash
set -a
source .env
set +a
python -m bible_bot
```

SQLite-файл по умолчанию создаётся в `runtime/bible_bot.db`. Токен и runtime-
данные исключены из Git.

## Docker

```bash
cp .env.example .env
# заполнить BOT_TOKEN
docker compose up -d --build
docker compose logs -f
```

Контейнер перезапускается автоматически, а каталог `runtime/` подключён как
именованный Docker volume.

## Как работает доставка

- Планировщик работает в том же процессе, что и long polling, и проверяет
  очередь каждые 30 секунд.
- В базе хранится точное следующее время отправки в UTC; пользовательское время
  вычисляется по IANA-зоне (`Europe/Minsk`, а не фиксированному `UTC+3`).
- Уникальная запись `(chat_id, local_date, kind)` защищает от повторной
  ежедневной отправки после перезапуска.
- Если Telegram временно недоступен, попытка откладывается на 10 минут; при
  rate limit учитывается `retry_after`.
- Если пользователь заблокировал бота, его подписка отключается.
- После длинного простоя старые стихи не высылаются пачкой: бот продолжает с
  актуального дня.

## Структура

```text
bible_bot/
  app.py          запуск Telegram polling и планировщика
  handlers.py     онбординг, настройки, избранное и циклы
  scheduler.py    ежедневная доставка и защита от повторов
  database.py     SQLite-хранилище
  content.py      чтение корпуса и выбор отрывков
  data/           локальный Новый Завет и планы
scripts/
  build_bible_data.py
tests/
```

## Обновление корпуса

Сгенерированные JSON-файлы уже находятся в репозитории. Для воспроизводимого
обновления:

```bash
python scripts/build_bible_data.py
```

Источник и условия использования описаны в [NOTICE.md](NOTICE.md). Текст
Синодального перевода в исходном наборе отмечен как Public Domain.
